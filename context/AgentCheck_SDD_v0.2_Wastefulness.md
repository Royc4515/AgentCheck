AgentCheck: Check #2 — Is It Wasteful?Software Design Document (SDD) v0.2 May 2026   1. OverviewWhile Check #1 focuses on whether an agent works, Check #2 audits the cost of operation. It identifies inefficiencies such as oversized context windows, unnecessary tool calls, and the use of overqualified models for simple tasks. This check transforms abstract "expensive feelings" into actionable data regarding tokens, dollars, and latency.  1.1 GoalsCost Measurement: Track cost-per-task in tokens, USD, and latency.  Pattern Identification: Pinpoint specific waste, such as bloated prompts containing unused examples.  Benchmarking: Compare agent performance against an efficient baseline implementation.  Pipeline Efficiency: Reuse the execution pipeline from Check #1 to avoid redundant runs.  1.2 Non-GoalsAuto-fixing: The tool reports waste; it does not automatically rewrite code.  Production Monitoring: This remains a pre-deployment audit tool.  Carbon Tracking: While considered, it is not currently in scope.  2. The Five Dimensions of WasteAgentCheck decomposes "wastefulness" into five measurable categories:  DimensionMeasurement LogicToken bloatCompares input/output tokens against the theoretical minimum; detects bloated prompts and repeated instructions.Model over-specDetermines if a task solved by a frontier model (e.g., GPT-4o) could be solved identically by a cheaper model (e.g., GPT-4o-mini).Tool-call churnAnalyzes the tool-call graph to find redundant or identical invocations within a single task.Retry stormsDetects more than two retries without backoff or retries on deterministic failures.Latency dragIdentifies independent tool calls that are executed sequentially instead of in parallel.2.1 Wastefulness ScoreThe tool outputs a score from 0 to 100, where 0 is perfectly efficient and 100 is catastrophic (100x baseline cost or worse).  Formula: $waste\_score = 100 - (baseline\_cost / actual\_cost) \times 100$   3. Architecture & Data FlowCheck #2 integrates into the existing v0.1 pipeline, adding one new component: the Waste Analyzer.  3.1 New Component: Waste AnalyzerThis component consumes raw execution logs to produce a structured report via five sub-components:  Token Profiler: Flags any prompt where tokens exceed 2x the baseline.  Model Downgrade Tester: Swaps models to see if cheaper tiers maintain a pass rate of $\ge 70\%$.  Tool-Call Graph Builder: Detects duplicate calls and finds parallelization opportunities.  Retry Detector: Pattern-matches unnecessary retry sequences.  Cost Estimator: Maps token usage to USD using a weekly-refreshed pricing table.  3.2 Pricing InfrastructureDynamic Fetching: Attempts to fetch the latest pricing.yaml from an Anthropic-maintained GitHub gist.  Offline Fallback: Uses bundled "last-known-good" data if no connection is available.  Snapshotting: Reports always display the price snapshot date to account for provider volatility.  4. The Model Downgrade LadderThe tool proof-tests cheaper models using a hardcoded "ladder":  OpenAI: GPT-4o $\rightarrow$ GPT-4o-mini $\rightarrow$ GPT-3.5-Turbo.  Anthropic: Claude Opus $\rightarrow$ Claude Sonnet $\rightarrow$ Claude Haiku.  Google: Gemini Pro $\rightarrow$ Gemini Flash $\rightarrow$ Gemini Flash-Lite.  To minimize audit costs, the downgrade test only runs on a sample of up to 5 tasks and is skipped if the agent's total cost is already below $0.01/task.  5.
